# Turtle Soup AI 개선 계획

> 구현 상태: P0, P1, P2 항목 구현 완료. 배포 전 신규 migration 적용과 Supabase 익명 로그인을 활성화해야 한다.

## 목표

현재의 핵심 게임 흐름을 유지하면서 서비스 운영 안정성, 사용자 기록 보존,
AI 문제 품질, 게임 지속성을 순서대로 개선한다. 기존 비로그인 사용자의 플레이
흐름과 공개 API 호환성을 최대한 유지하며 단계적으로 적용한다.

## 현황

- 오늘의 문제, 이전 문제 선택, AI 문답, 정답 제출, 포기 및 세션 연장이 구현되어 있다.
- 한 세션은 기본 20분이며 유효 질문은 최대 30개로 제한된다.
- 게임 기록은 브라우저 `localStorage`에 저장된 `deviceId`를 기준으로 조회한다.
- 문제 자동 생성, AI 품질 검수, 예약 공개 및 생성 이력 관리가 구현되어 있다.
- 결과 공유와 기본 통계는 제공하지만 문제 평가, 신고, 힌트 기능은 없다.
- 애플리케이션 수준의 요청 제한과 구조화된 운영 관측 기능은 부족하다.
- 현재 테스트 55개와 ESLint 검사는 통과한다.

## 우선순위별 개선 사항

### P0: 안정성 및 비용 보호

- `/api/chat`, `/api/answers`, 세션 생성 및 연장 API에 IP와 `deviceId` 기반 요청 제한을 적용한다.
- 각 API 요청의 요청 ID, 처리 시간, 응답 상태와 OpenAI 오류 유형을 구조화 로그로 기록한다.
- AI 요청 실패 시 사용자가 작성한 입력을 보존하고 명시적인 재시도 동작을 제공한다.
- 세션 연장은 세션당 1회로 제한하고 서버에서 횟수를 검증한다.
- 정답 제출, 포기, 세션 연장의 조건부 업데이트 결과를 검사하여 동시 요청에 의한 상태 덮어쓰기를 방지한다.

### P1: 기록 영속성과 사용자 경험

- 선택적 Supabase 익명 로그인을 도입하고 로그인 이후 생성되는 기록을 계정에 귀속한다. 기존 `deviceId` 기록은 보안상 자동 이전하지 않는다.
- 로그인하지 않은 사용자는 현재 방식으로 계속 플레이할 수 있게 하되 기록 손실 가능성을 안내한다.
- 진행 중인 세션이 있으면 홈 화면에 해당 문제의 `이어하기` 동작을 표시한다.
- 네트워크 오류, 일일 문제 미등록, 기록 없음 화면에 재시도 또는 적절한 이동 동작을 제공한다.
- 아카이브에 상태, 난이도, 카테고리 필터와 문제명 검색을 추가한다.

### P1: 문제 품질 피드백

- 종료 화면과 기록 상세 화면에 문제 평가 기능을 추가한다.
- 평가는 재미, 난이도 적절성, 정답 타당성과 선택적 의견으로 구성한다.
- 문제 신고 사유로 정답 오류, 모호함, 불쾌한 내용, 저작권 우려를 지원한다.
- 낮은 평가나 신고가 누적된 문제를 운영자가 확인하고 공개 중단할 수 있는 상태와 운영 쿼리를 제공한다.
- 문제별 사용자 평가 집계를 보존하여 향후 AI 자동 생성 및 검수 정책 개선에 활용한다.

### P2: 게임성 확장

- 질문 10개와 20개 사용 시 선택형 힌트를 사용할 수 있게 한다.
- 힌트는 문제 생성 단계에서 함께 생성하고 검수하며 정답을 직접 노출하지 않아야 한다.
- 힌트 사용 여부와 횟수를 세션에 기록하고 결과 통계 및 공유 문구에 반영한다.
- 기록 화면에 연속 참여 일수, 해결률, 카테고리별 성적과 평균 질문 수를 추가한다.
- 공유 결과에는 문제 제목이나 정답 대신 질문 수, 소요 시간, 힌트 사용량과 플레이 URL만 포함한다.

### P2: 접근성 및 화면 품질

- 모든 `dialog`에 제목 연결, 초기 포커스, 닫기 및 포커스 복귀 동작을 적용한다.
- 타이머와 질문 수 변경이 스크린 리더에 과도하게 반복되지 않도록 알림 빈도를 제한한다.
- 색상뿐 아니라 아이콘과 텍스트로 상태를 구분하고 작은 화면에서 헤더 배치를 보완한다.
- 한국어 UI에 혼재된 영문 상태 문구와 카테고리 표기를 일관되게 정리한다.

## 데이터 및 API 변경

### 데이터베이스

- `game_sessions`에 `user_id`, `extension_count`, `hint_count`를 추가한다.
- `problems`에 단계별 힌트와 `moderation_status`를 추가한다.
- 문제 평가와 신고를 저장하는 `problem_feedback` 테이블을 추가한다.
- `problem_feedback`에는 문제, 세션, 사용자 또는 기기, 평가 항목, 신고 사유,
  선택적 의견과 생성 시각을 저장한다.
- 사용자별 통계와 문제별 평가 집계에 필요한 인덱스를 추가한다.

### API 및 타입

- 세션 응답에 `extensionCount`, `hintCount`와 사용 가능한 힌트 단계를 추가한다.
- 힌트 조회, 문제 평가 제출, 사용자 통계 조회 API를 추가한다.
- 모든 쓰기 API에 공통 요청 제한 정책과 일관된 구조화 오류 형식을 적용한다.
- 기존 `deviceId` 필드는 비로그인 호환성을 위해 유지하고 인증 사용자는 서버에서 `userId` 소유권도 검증한다.
- 공개 중단된 문제는 일일 문제와 문제 목록 API에서 제외한다.

## 테스트 기준

- 요청 제한의 허용 범위, 초과 응답과 일정 시간 후 복구를 검증한다.
- 동시 정답 제출, 포기 및 연장 요청이 하나의 최종 상태만 만들도록 검증한다.
- 세션 연장 횟수 제한과 만료 세션 처리 정책을 API 테스트로 검증한다.
- 인증 이후 기록의 계정 귀속과 기존 기기 기록의 비로그인 호환성을 검증한다.
- 힌트 해제 조건, 중복 사용, 질문 제한과의 상호작용을 검증한다.
- 평가 중복 제출, 신고 누적과 공개 중단 문제의 사용자 API 제외를 검증한다.
- AI 실패 후 입력 보존 및 재시도 흐름을 컴포넌트 테스트로 검증한다.
- 모바일 화면과 키보드 전용 `dialog` 흐름을 브라우저 테스트로 확인한다.
- 각 단계 완료 시 `npm test`, `npm run lint`, `npm run build`가 모두 통과해야 한다.

## 적용 순서

1. 요청 제한, 구조화 로그, 동시 상태 변경 보호 등 P0 안정성 개선
2. 선택적 인증, 인증 이후 기록 귀속과 홈 화면 이어하기
3. 문제 평가, 신고 및 공개 중단 운영 흐름
4. 단계별 힌트와 확장 통계
5. 접근성 검증과 UI 문구 및 반응형 화면 정리

리더보드는 계정 체계와 부정 사용 방지가 안정된 이후 과제로 제외한다. 힌트는
자동으로 노출하지 않고 사용자가 명시적으로 요청할 때만 사용 횟수에 반영한다.

# AI Prompt Policy

## Game master

- 하나의 사실을 확인하는 폐쇄형 질문만 허용
- 출력은 `yes | no | irrelevant | invalid_question` 구조화 결과
- 설명, 힌트, 정답 일부를 노출하지 않음
- 개방형 정답 요구에는 지정된 안내 문구 반환

## Answer validation

- 문구 일치가 아니라 핵심 원인과 인과관계를 평가
- 정답과 양립하는 상상적 세부사항, 동의어, 의역 허용
- 실제 정답과 모순되거나 핵심 메커니즘이 다르면 오답
- 정답 판정에는 `isCorrect=true`와 `confidence >= 0.7` 모두 필요

## Problem generation

- `SCRAPE_SOURCE_URLS`에서 수집한 공개 웹 문제는 참고 자료로만 사용
- 웹 참고 문제를 사용할 때도 원문 문장과 전개를 그대로 저장하지 않고 서비스용 문안으로 재구성
- 웹 참고 생성 결과는 `source='Web'`, `source_url`로 출처 기록
- 웹 참고 문제가 없으면 AI 순수 창작으로 fallback하지 않고 생성 실패 처리
- 실제 바다거북 스프와 black stories식 추리 구조를 참고하되, black stories 상용 카드 문구와 고유 사건은 복제 금지
- 현실적인 2~4단계 인과관계
- 관찰 가능한 단서 최소 두 개
- 꿈, 연극, 가상 화면, 말장난만으로 만든 반전 금지
- 영어 단어의 다의성, 동음이의어, 발음, 철자, 약어, 알파벳, 번역 차이가 핵심인 문제 금지
- `shoot/shot` 사진 말장난, NATO 음성 알파벳, Scrabble, AA/16진수, HTTP 상태 코드처럼 한국어로 옮기면 핵심이 무너지는 장치 금지
- 카드게임 하트, 포커, 체스, Monopoly, 특정 국가·문화·스포츠 관습처럼 외부 규칙을 알아야만 풀리는 문제 금지
- 본문에 명확히 공개된 상황 규칙만으로 한국어 플레이어가 추론할 수 있는 경우만 허용
- 본문에 없는 희귀 질환, 특수 직업, 임의 규칙 의존 금지
- 비그래픽하고 서비스에 안전한 내용
- 기존 최대 300개 문제의 제목과 상황을 중복 검사에 제공

생성 후보는 별도 검수 호출에서 인과성, 공정성, 중복, 한국어 품질, 안전성, 웹 참고 원문 복제 여부를 평가하며 80점 이상만 저장합니다.
